## 三、语音识别工作流

### 3.1 整体流程

```
用户说话 → 麦克风录音 → ARM端录制PCM → 发送给x86 Ubuntu
                                              │
                                              ▼
                                   讯飞MSC语音识别引擎
                                   根据cmd.bnf语法匹配
                                              │
                                              ▼
                                   返回XML（含cmd id）
                                              │
                                              ▼
ARM端解析XML → 根据id执行对应功能
```

### 3.2 BNF 语法文件 (`robot/x86/bin/cmd.bnf`)

这是语音识别的"词典"，定义了用户能说什么，以及对应的命令ID。

```
# 格式：语音文本!id(数字)

<cmd>:
  你好!id(2)           # 打招呼
  量血压!id(3)         # 量血压 → 触发 mks-all
  测心率!id(6)         # 测心率 → 触发 mks-hr
  测血氧!id(7)         # 测血氧 → 触发 mks-spo2
  体温!id(8)           # 测体温 → 触发 asm-temp
  健康报告!id(10)      # 综合检测 → mks-all + asm-temp
  帮助!id(0)           # 帮助菜单
  你是谁!id(66)        # 自我介绍
  再见!id(999)         # 退出系统
```

### 3.3 如何添加新语音指令

**步骤1：编辑 `cmd.bnf`**

在对应 `<cmd>` 区域添加新条目：
```
# 例如：增加"测量身高"指令
测量身高!id(11)|测身高!id(11)
```

**步骤2：将新 `cmd.bnf` 部署到 x86 Ubuntu**
```bash
# 在 x86 Ubuntu 上，将 cmd.bnf 放到 main 程序同目录
cp cmd.bnf /path/to/x86/bin/
# 重启识别引擎
./main
```

**步骤3：在 ARM 端 `voicectl.c` 中处理新命令**
```c
if (cmd == 11) // 新增功能
{
    // 你的处理逻辑
}
```

## 四、ARM ↔ STM32 串口通信协议

### 4.1 物理连接

```
ARM GEC210  ←→  STM32F407
  串口2         串口3 (PB10/PB11)
 /dev/ttySAC2    USART3
   波特率: 115200, 8N1
```

### 4.2 命令表

| ARM 发送 | STM32 返回 | 超时 | 说明 |
|----------|-----------|------|------|
| `mks-all\n` | `HR,SBP,DBP,SPO2\n` 例:`72,118,75,98\n` | ~80s | 一次充气获取全部 |
| `mks-hr\n` | `72\n` | ~50s | 仅心率 |
| `mks-spo2\n` | `98\n` | ~50s | 仅血氧 |
| `asm-temp\n` | `36.5\n` | ~30s | 体温(℃) |

> **注意**：`mks-sbp`、`mks-dbp` 和 `mks-bp` 已被 `mks-all` 替代，不再使用。
> 如需单独测某指标仍用 `mks-hr` 或 `mks-spo2`（不充气、更快）。

### 4.3 通信时序

```
ARM端 get_stm32_mksall() / get_stm32_data() :
  1. tcflush() 清空串口缓冲
  2. write() 发送命令字符串
  3. 每秒轮询 read() 检查 STM32 回复
  4. 如果 STM32 返回 0：传感器还没就绪 → 重新发命令
  5. 超时后返回失败

STM32端 main.c :
  1. 检测 USART3_RX_STA 标志（收到完整一行）
  2. 匹配命令字符串
  3. 调用 Get_MKS141_Date() 等待传感器数据
  4. sprintf() 格式化数据 → Usart_SendString() 发回
```

### 5.3 文本框布局详解 (`font_show` 函数)

**函数签名：**
```c
void font_show(
    char *text,           // 要显示的文字（支持中文）
    int pixelSize,        // 字体大小（磅数），如 24, 28, 32, 38, 40
    int outFrameWidth,    // 文本框宽度（像素）
    int outFrameHeight,   // 文本框高度（像素）
    int outFrameColor,    // 文本框背景色（0xAARRGGBB 格式）
    int fontPosX,         // 文字在文本框内的 X 坐标
    int fontPosY,         // 文字在文本框内的 Y 坐标
    int fontColor,        // 文字颜色（0xAARRGGBB 格式）
    int frameToLcdPosX,   // 文本框放置在 LCD 上的 X 坐标
    int frameToLcdPosY    // 文本框放置在 LCD 上的 Y 坐标
);
```

**参数图解：**
```
  LCD坐标系 (0,0) 左上角，800×480
  ┌──────────────────────────────────────┐
  │                                      │
  │     frameToLcdPosX →                 │
  │     frameToLcdPosY →┌─────────────┐ │
  │                      │ 文本框      │ │ ← outFrameWidth × outFrameHeight
  │                      │ outFrameColor 背景填充
  │                      │             │ │
  │                      │ fontPosX,fontPosY → "文字内容" (fontColor) │ │
  │                      └─────────────┘ │
  │                                      │
  └──────────────────────────────────────┘
  LCD 全屏: WIDTH=800, HEIGHT=480
```

**常用尺度速查表：**

| 场景 | pixelSize | outFrameW | outFrameH | fontPosX | fontPosY | 适合字数 |
|------|-----------|-----------|-----------|----------|----------|---------|
| 大标题 | 38~40 | 700 | 60 | 10 | 5 | ≤8字 |
| 单行提示 | 32 | 700 | 80 | 10 | 10 | ≤12字 |
| 单行提示(小) | 28 | 700 | 50 | 10 | 5 | ≤14字 |
| 小标签（左栏） | 28 | 150 | 50 | 10 | 5 | ≤5字 |
| 小标签（右栏） | 28 | 150 | 50 | 10 | 5 | ≤5字 |
| 多行文本 | 24~26 | 700 | 350 | 10 | 10 | ≤40字 |
| 退出大字 | 40 | 430 | 147 | 120 | 45 | ≤5字 |

**调整原则：**
1. **文本要完整显示** → 如果文字被截断，增大 `outFrameWidth` 或减小 `pixelSize`
2. **文本框不能超出屏幕** → `frameToLcdPosX + outFrameWidth ≤ 800`，`frameToLcdPosY + outFrameHeight ≤ 480`
3. **左右双栏布局** → 左栏 `frameToLcdPosX=10`，右栏 `frameToLcdPosX=170`（宽150，间隔10px），每栏高50，间距10px
4. **背景色透明/半透明** → `outFrameColor` 用 `0x80xxxxxx`（有Alpha通道时）
5. **文字颜色** → 常用 `0xff000000`（黑）、`0xffff0000`（红）、`0xff0000ff`（蓝）

**常见调整示例：**
```c
// 示例1：把"帮助菜单"里某条文本改大
// 原来：font_show("\"你好\" -- 打招呼", 28, 150, 50, 0xff00ff00, 10, 5, 0xff000000, 10, 60);
// 改为大字号：需同步增大框体
font_show("\"你好\" -- 打招呼", 32, 160, 60, 0xff00ff00, 10, 5, 0xff000000, 10, 60);
//                                  ↑字体  ↑框宽 ↑框高  （框要装得下更大文字）

// 示例2：把测量结果往下挪
// 原来：font_show(font_data, 32, 700, 300, 0xffffff00, 10, 80, 0xff000000, 36, 26);
// 往下移50px：
font_show(font_data, 32, 700, 300, 0xffffff00, 10, 80, 0xff000000, 36, 76);
//                                                                    ↑Y+50

// 示例3：多行报告文本换行
// 用 \n 即可换行，字体别太大，框要够高
char report[512];
sprintf(report, "==== 健康报告 ====\n收缩压：%d mmHg\n舒张压：%d mmHg\n心率：%d 次/分", sbp, dbp, hr);
font_show(report, 24, 700, 350, 0xffffff00, 10, 10, 0xff000000, 36, 26);
//                 ↑小字 ↑宽  ↑够高才能显示多行
```

## 六、STM32 端代码说明 (`stm32/project(1)/USER/main.c`)

### 6.1 核心逻辑

```c
// 主循环：
while (1)
{
    if (USART3_RX_STA)  // 收到 ARM 命令
    {
        // 匹配命令字符串
        if (contains "asm-temp")  → 测体温 → 回复 "36.5\n"
        if (contains "mks-all")   → 测全部 → 回复 "72,118,75,98\n"
        if (contains "mks-hr")    → 测心率 → 回复 "72\n"
        if (contains "mks-spo2")  → 测血氧 → 回复 "98\n"
        // 未匹配 → 忽略
    }
}
```

### 6.2 传感器驱动

| 文件 | 功能 |
|------|------|
| `HARDWARE/pressure/mks141.c` | MKS141 血压/心率/血氧一体模块 |
| `HARDWARE/temperature/as6221.c` | AS6221 I2C 温度传感器（偏差+3℃补偿） |

### 6.3 如何添加新传感器命令

在 `main.c` 的 while 循环中添加 `else if` 分支：

```c
// 示例：添加"环境光"传感器
else if (contains_substring((uint8_t *)cmd_buf, "light"))
{
    uint16_t lux = BH1750_Read();  // 你的传感器读取函数

```

## 八、常见问题

### Q1: 系统不启动？
```bash
# ARM 端检查
dmesg | grep -i fb0       # LCD 是否正常
dmesg | grep -i input     # 触摸屏是否正常
dmesg | grep -i ttySAC2   # 串口是否识别
ls /dev/fb0 /dev/input/event0 /dev/ttySAC2
```

### Q2: 语音识别没反应？
- 检查 x86 Ubuntu 上 `main` 是否在运行（端口 54321）
- 检查 ARM 和 Ubuntu 能否 ping 通
- 检查 `cmd.bnf` 是否正确部署

### Q3: 传感器返回全0？
- MKS141 需要人手指/手腕放入传感器
- 检查 STM32 串口3线是否接好（PB10/PB11 ↔ ARM 串口2）
- ARM 端 `printf` 会打印 STM32 原始回复，查看是否正确

### Q4: 声音不播放？
```bash
# ARM 端测试
aplay /ai_wav/welcome.wav
# 如果报设备忙，先kill其他aplay进程
killall aplay
```
